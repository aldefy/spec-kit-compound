00

Why this exists

LLMs are built to please. As context grows they affirm, drift, and quietly expand scope — a "while I'm in here, let me also…" reflex that turns a focused change into a sprawling one. Raw prompting has no checkpoints, so nothing stops the slop before it lands.

Spec-driven development adds checkpoints; SpecKit brings it to coding agents. spec-kit-compound adds the missing discipline: lock the boundary before planning, complete the obligation set, and hand judgment to a different model — once for the plan (planverify), once for the diff (intentguard). It's an extension, not a harness — it rides on Claude Code, Codex, Cursor, or Gemini CLI.

01

ICE — lock the boundary first

An interview captures three things and freezes them, so drift has something measurable to be measured against.

I
Intent

What to build — and explicitly what not to touch. In-scope and out-of-scope, declared up front.

C
Context

The why, how, and where — dependencies and contracts. Ambiguity removed before design starts.

E
Expectation

Scenarios and hard boundaries in a builder-hidden file — compartmentation against teaching to the test.

02

Compound engineering memory

A local store of ADRs, corrections, and patterns — decisions made, mistakes hit, fixes that worked. It persists across features, so learnings carry forward instead of evaporating with the context window. Committed to git: a teammate's LLM reads the same hard-won context. Every feature writes back what it learned — the system compounds.

04

The firewall — why a stranger judges

Whoever did the work is its worst judge — motivated reasoning, plus they remember what they meant. So each gate seals a briefing of only the locked criteria + the artifact + mechanical facts, and hands it across a wall to a checker that never saw the work.

06

Enforcement — opt-in, cross-vendor

  • Gates are advisory by default. SKC_PLANVERIFY_GATE=block turns the teeth on.
  • A PreToolUse hook — working under both Claude Code and Codex CLI (the converged exit-2 contract) — blocks the first source write when the verdict is missing or BLOCKED_DRIFT. Doc and spec writes are never blocked.
  • A live localhost dashboard shows the whole chain — every stage, the planverify drift panel, file diffs, token spend — read-only, zero dependencies.
